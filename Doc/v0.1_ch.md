## 项目概述

本项目是一个基于 Unity 的策略游戏 POC，主题为全球数字化扩张。

长期概念是一个由格子划分的球形世界。玩家建立总部，通过相邻格子扩张，建造基础设施，发展科技，争夺全球影响力。

当前阶段严格只是**概念验证（Proof of Concept）**。现在不要实现完整的游戏设计。

## 当前 POC 范围

POC 只包含两个核心功能：

1. 一个可旋转的、几乎全由六边形网格覆盖的球形世界。
2. 玩家选择一个总部位置，并向相邻格子扩张。

未经明确要求，不要添加以下内容：

- 经济系统
- 科技树
- 硬件、网络或软件阵营
- AI 对手
- 多人联机
- 国界
- 真实地形
- 资源系统
- 建筑升级
- 战斗公式
- 商业化
- 存档系统
- DOTS/ECS
- Addressables
- 后端服务

POC 的目的是验证球面领土选择与扩张是否易于理解、是否有趣。

## 核心架构规则

### 旋转球体，而不是相机

主相机在结构上保持稳定，由 `GlobeRoot` 绕世界原点旋转。

原因：

- 格子选择更容易推理。
- 球体本地坐标保持稳定。
- 建筑和特效一起旋转。
- 纬度、格子查找和邻接关系与视觉旋转互相独立。

相机可以改变距离用于缩放，但在 POC 阶段不应独立自由环绕。

### 分离数据与表现

游戏逻辑不能只存储在 MonoBehaviour 组件里。

格子用纯 C# 数据类，渲染与交互用独立的视图组件。

推荐的分离：

```text
GlobeCellData
- 纯游戏数据
- 无 MonoBehaviour

GlobeCellView
- 渲染
- 悬停
- 选择
- 颜色更新
- 扩张动画
```

### 不要过早优化

对于 POC，约 642 个格子时，每个格子一个 GameObject 是可接受的。

不要一开始就用：

- 合并式程序化网格选择系统
- GPU 驱动渲染
- 计算着色器
- ECS
- 自定义渲染通道
- 复杂的对象池系统

只有在 POC 交互被验证之后，才开始优化。

## 球面网格规则

球面无法只用正六边形铺满。

所需的拓扑是：

```text
大量六边形
+ 正好 12 个五边形
```

这 12 个五边形是球面拓扑在数学上必需的。无论细分级别如何，其数量恒为 12。

### 生成方法

采用如下流程：

```text
二十面体
→ 细分三角形
→ 将顶点归一化到球面
→ 构建 Icosphere
→ 生成对偶网格
→ Icosphere 顶点成为游戏格子
→ 相邻的 Icosphere 顶点成为相邻格子
→ 顶点周围三角形的中心成为格子多边形的角点
```

大多数 Icosphere 顶点有六个相邻三角形，因此生成六边形。

原始的 12 个拓扑顶点有五个相邻三角形，因此生成五边形。

### 推荐细分级别

对于 POC，使用细分级别 3：

```text
约 642 个格子
12 个五边形
约 630 个六边形
```

细分级别 2（约 162 个格子）适合更早期的技术测试。

### 五边形用途

五边形是总部候选格子。

初始规则：

```text
六边形：
- 普通领土格子

五边形：
- 总部候选
- 战略核心节点
```

不要在生成器中把每个五边形永久硬编码为总部。

网格生成器只报告多边形边数或格子类型。游戏逻辑代码决定一个五边形当前是：

- 候选点
- 总部
- 中立战略节点
- 特殊目标

推荐字段：

```text
CellType
- Hexagon
- Pentagon

CellRole
- Normal
- BaseCandidate
- Headquarters
```

### 五边形玩法属性

五边形有五个邻居，而不是六个。

这自然使它：

- 略微更易防守
- 在其他条件相同的情况下，从它扩张略微更慢

对于 POC，保持这个属性不变。

推荐的初始总部行为：

```text
玩家选择一个五边形。
该五边形成为总部。
其五个相邻格子可选地成为初始拥有领土。
```

第一版实现可以只让五边形本身被拥有，之后再添加第一环的拥有关系。

## 格子数据模型

最小推荐的格子数据：

```text
GlobeCellData
├── Id
├── CellType
├── CenterDirection
├── PolygonVertices
├── NeighborIds
├── OwnerId
├── TerritoryState
└── ExpansionProgress
```

定义：

```text
Id
- 唯一格子标识

CellType
- 六边形或五边形

CenterDirection
- 从球心到格子中心的单位向量

PolygonVertices
- 本地空间的多边形边界顶点

NeighborIds
- 相邻格子的 Id

OwnerId
- -1 表示中立，否则为玩家 ID

TerritoryState
- Neutral
- Expanding
- Owned
- Headquarters

ExpansionProgress
- 归一化的 0 到 1 进度值
```

选择是一种视觉状态，不应成为领土所有权状态的一部分。

推荐的视觉状态：

```text
CellVisualState
- Normal
- Hovered
- Selected
```

总部也可以被选中，因此逻辑状态与视觉状态必须保持分离。

## 球体交互

期望的交互类似虚拟轨迹球（trackball / arcball）式的球体操作。

### 初始实现

使用屏幕空间的鼠标位移映射到相机空间的轴：

```text
水平鼠标移动
→ 绕相机 up 轴旋转 GlobeRoot

垂直鼠标移动
→ 绕相机 right 轴旋转 GlobeRoot
```

使用四元数，而不是存储的欧拉 yaw 与 pitch 值。

存储：

```text
targetRotation: Quaternion
```

避免依赖：

```text
yaw: float
pitch: float
```

原因：

- 避免极点限制
- 避免欧拉角环绕
- 避免万向锁相关行为
- 允许全方向旋转
- 便于后续替换为 arcball

旋转轴必须来自相机：

```text
camera.transform.up
camera.transform.right
```

拖动交互不要使用固定世界轴。

### 旋转顺序

将新的相机空间旋转应用在当前球体旋转之前，使拖动方向在视觉上保持一致。

### 拖动与点击的区分

输入必须区分：

```text
PointerDown
PotentialClick
Dragging
PointerUp
Click
```

使用一个小的屏幕空间阈值，约 5 到 10 像素。

行为：

```text
指针按下
→ 记录起始位置

移动低于阈值
→ 仍是潜在点击

移动高于阈值
→ 开始球体拖动

拖动后指针释放
→ 应用惯性

无拖动的指针释放
→ 选择格子
```

### 惯性

拖动结束时，球体应继续短暂旋转。

使用帧率无关的指数阻尼：

```text
velocity *= exp(-damping * deltaTime)
```

不要每帧减去固定量。

### 缩放

通过改变相机与球心的距离来缩放。

不要缩放球体。

不要把 FOV 作为主要缩放控制。

推荐行为：

```text
鼠标滚轮
→ 更新目标相机距离
→ 在最小与最大距离之间钳制
→ 平滑插值实际距离
```

建议的初始相机参数：

```text
Projection: Perspective
Field of View: 45
Near Clip Plane: 0.1
Far Clip Plane: 500
```

## 格子选择

使用从相机穿过指针位置的射线检测。

流程：

```text
屏幕指针位置
→ 相机射线
→ 对可见格子碰撞体进行射线检测
→ 解析出 GlobeCellView
→ 读取 CellId
→ 更新悬停或选择
```

背面的格子不可被选择。

在 POC 阶段，每个格子可以有自己的碰撞体。

选择规则：

```text
悬停中立或已拥有的格子
→ 应用悬停高亮

无拖动地点击格子
→ 设置 SelectedCell

点击空白区域
→ 清除选择
```

不要让 UI 交互旋转或选择球体。遵守 EventSystem 的“指针是否在 UI 上”检查。

---

## 渲染规则

推荐的 POC 视觉栈：

```text
深色底球
+ 略微抬高的多边形格子
+ 可见的格子边框
+ 中立灰色格子
+ 玩家配色的已拥有格子
+ 更亮的总部格子
+ 扩张时的脉冲或填充动画
```

用略微不同的半径避免 Z 冲突（Z-fighting）。

示例：

```text
底球半径：10.00
格子表面半径：10.02
边框半径：10.03
总部标记半径：10.10
```

POC 阶段可接受低多边形（low-poly）外观。

现在不要投入精力做写实地球渲染、大气、云、国家或地形。

## 总部放置

游戏开始时：

```text
将全部 12 个五边形格子高亮为总部候选。
```

玩家流程：

```text
选择一个五边形
→ 显示放置确认
→ 确认总部
→ 设置 OwnerId
→ 将 TerritoryState 设为 Headquarters
→ 生成一个简单的总部标记
→ 关闭所有其它候选高亮
```

对于 POC，总部标记可以是：

- 圆柱体
- 圆锥体
- 自发光球体
- 简单图标

使用格子中心方向放置：

```text
position = globeCenter + centerDirection * markerRadius
up = centerDirection
```

标记必须保持为 `GlobeRoot` 的子物体。

只能创建一个初始总部。

## 扩张规则

第一版扩张实现应由玩家主导。

### 基本规则

玩家只能扩张到与其已拥有领土相邻的中立格子。

校验：

```text
目标格子为中立
且
至少一个 NeighborId 属于该玩家
且
该目标尚无正在进行的扩张
```

### 初始扩张流程

```text
玩家选择相邻的中立格子
→ 扩张开始
→ 格子进入 Expanding 状态
→ 播放视觉进度
→ 固定时长后，OwnerId 变为该玩家
→ 格子进入 Owned 状态
→ 前沿更新
```

建议的初始时长：

```text
每格 1 到 2 秒
```

### 扩张并发

第一版实现使用一个扩张槽位。

不要自动从每个边界格子扩张，因为那会造成指数级的领土增长。

推荐的初始规则：

```text
总部提供一个活动扩张线程。
```

后续系统可以增加扩张槽位，但不在初始 POC 阶段。

### 可选的自动扩张

只有在手动扩张可用之后，可选模式才每次选择一个随机前沿格子。

第一遍不要实现定向扩张、阵营加成或复杂寻路。

---

## 前沿定义

前沿格子是至少有一个邻居被玩家拥有的中立格子。

概念流程：

```text
对每个已拥有格子
→ 检查 NeighborIds
→ 若邻居为中立
→ 将该邻居加入前沿集合
```

用集合避免重复的前沿条目。

不要每帧扫描每个格子。只在所有权变化时重新计算，或增量维护前沿。

对于 642 个格子的 POC，每次扩张后完整重算仍然可接受。

## 推荐的脚本职责

### GlobeGenerator

职责：

- 创建基础二十面体
- 细分三角形
- 将顶点归一化到球半径
- 构建 Icosphere 拓扑
- 生成对偶的五边形与六边形格子
- 一致地排序多边形角点
- 构建邻居列表
- 实例化或提供格子视图

### GlobeCellData

职责：

- 存储纯格子数据
- 无渲染逻辑
- 无输入处理
- 不继承 MonoBehaviour

### GlobeCellView

职责：

- 渲染格子网格
- 应用中立、已拥有、总部、悬停、选择等视觉
- 显示扩张进度
- 暴露关联的 CellId

### GlobeInputController

职责：

- 处理指针输入
- 区分点击与拖动
- 旋转 GlobeRoot
- 应用惯性
- 处理缩放
- 执行射线检测
- 忽略 UI 上方的指针输入

### BasePlacementController

职责：

- 追踪已选候选
- 校验总部放置
- 创建总部
- 初始化玩家所有权
- 阻止第二个初始总部

### ExpansionController

职责：

- 校验邻接
- 开始扩张
- 追踪活动扩张槽位
- 完成所有权变更
- 重算前沿

### GameManager

职责：

- 持有 POC 级别的游戏状态
- 存储玩家 ID
- 协调放置与扩张系统
- 暴露已拥有格子的统计

避免把 GameManager 变成一个包揽一切的实现类。

## 开发顺序

除非遇到阻塞性问题需要调整，否则按如下确切顺序实现：

```text
1. 创建并保存 GlobePOC 场景。
2. 构建 WorldRoot、GlobeRoot、CameraRig 和 Main Camera 层级。
3. 添加一个临时球体。
4. 实现球体拖动旋转。
5. 实现相机距离缩放。
6. 实现点击与拖动的区分。
7. 验证在临时球体上的射线命中位置。
8. 生成 Icosphere。
9. 生成对偶的五边形与六边形格子。
10. 验证正好 12 个五边形。
11. 验证正确的邻居数量。
12. 将格子显示为独立视图。
13. 实现悬停与点击选择。
14. 改变单个格子颜色。
15. 将五边形标记为总部候选。
16. 放置一个总部。
17. 允许扩张到一个相邻格子。
18. 添加扩张进度视觉。
19. 添加已拥有格子数量与百分比 UI。
20. 只有在手动扩张稳定之后，才添加可选的单线程自动扩张。
```

技术风险最高的任务是对偶网格生成和一致的多边形角点排序。

在格子拓扑被验证之前，不要开始游戏功能。

## 拓扑校验

网格生成后，断言或记录以下内容：

```text
五边形数量 == 12
每个五边形的邻居数 == 5
每个普通六边形的邻居数 == 6
每一条邻接关系都是双向的
无重复的格子 Id
一个格子内无重复的邻居 Id
无格子引用自身
所有多边形顶点都接近预期的球半径
```

如果某个格子多边形看起来扭曲或自交，说明该格子中心周围的三角形中心排序不正确。

在每个格子的本地切平面内排序多边形角点。

不要用全局 XYZ 坐标排序角点。

## POC 验收标准

### 球体交互

- 球体可以向任意方向旋转。
- 旋转不在极点锁定。
- 大角度旋转后拖动方向仍然视觉一致。
- 缩放平滑且被钳制。
- 点击不会被误判为拖动。
- UI 交互不会旋转球体。
- 背面的格子不可被选择。

### 网格

- 结果在视觉上是球形的。
- 大多数格子是六边形。
- 正好 12 个格子是五边形。
- 五边形与六边形的邻接关系正确。
- 每个格子可以被单独高亮。
- 每个格子可以独立改变所有者颜色。

### 总部

- 12 个五边形作为候选可见。
- 玩家可以选择一个五边形。
- 只能放置一个初始总部。
- 总部朝向遵循球面法线。
- 总部随 GlobeRoot 一起旋转。

### 扩张

- 只能扩张到相邻的中立格子。
- 扩张有可见进度。
- 扩张在固定时长后完成。
- 所有权颜色正确更新。
- 新拥有的格子成为有效的扩张来源。
- 第一版实现中同时只有一个扩张在进行。

### 统计

- 已拥有格子数量在扩张后更新。
- 领土百分比在扩张后更新。

## 编码规范

- 使用按功能分组的清晰命名空间。
- 每个类聚焦于单一职责。
- 优先使用序列化的私有字段，而非公开可变字段。
- 在合适处对基于校验的操作使用 `Try...` 方法。
- 避免在输入与扩张循环中的每帧分配。
- 避免在热更新路径中使用 LINQ。
- 扩张计时使用可安全取消的协程或显式状态。
- 在 `Awake` 或 `OnValidate` 中校验序列化引用。
- 记录可操作的错误，避免静默失败。
- 生成的网格数据保持在球体本地坐标系中。
- 世界旋转只放在 `GlobeRoot` 上。
- 面板可见的字符串必须使用英文。这包括 `Header`、`Tooltip`、`ContextMenu`、在 Inspector 中显示的枚举标签，以及自定义编辑器按钮文本。代码内部注释可以使用中文。

建议的命名空间：

```text
GlobalExpansion.Globe
GlobalExpansion.Input
GlobalExpansion.Gameplay
GlobalExpansion.UI
```

## 非目标与护栏

不要把 POC 变成写实的地理空间仿真。

不要把五边形当作必须隐藏的缺陷。它们是有意设计的战略节点。

不要尝试纯全六边形球体。在预期结构下，闭合球面铺砌在拓扑上是不可能的。

不要让每个格子同时扩张。

不要把悬停、选择、所有权和扩张绑进单一枚举。

在初始 POC 阶段，不要同时独立旋转相机与球体。

在基本交互被验证之前，不要构建正式的游戏平衡。

不要添加联网或持久化。

## 开发进度

最后更新：2026-07-15

### 已完成

- **球体交互**（`Assets/test.cs`，挂在 `GlobeRoot` 上）：
  - 使用相机空间的轴与四元数实现鼠标拖动旋转（无极点锁定、无万向锁问题）。
  - 松手惯性，采用帧率无关的指数阻尼。
  - 滚轮缩放通过改变相机距离实现（不缩放球体）；缩放范围由初始相机距离推算。
  - 相机保持初始设定的注视角度：正对球心的方向加上场景中设定的初始俯仰偏移（约 20°），而不是直接对准球心。
  - 注视俯仰随缩放阶段性调整：Inspector 滑块（`zoomTiltDelta`）让每个缩放步长增加相应度数的俯仰，并做钳制以避免在缩放极限处翻转。
  - 旋转由鼠标右键驱动：在任意位置（包括球体上）按住右键拖动即可旋转，松开后交给惯性。左键永远不旋转。
- **格子选择**（`test.cs` + `GlobeCellView`）：左键点击格子（松手时未超过像素阈值）会把该格颜色切换为选中色；再次点击恢复原色。背面格子被正面格子遮挡，不会被选中。指针在 UI 上时通过 EventSystem 忽略选择。
- **网格生成**（`Assets/GlobeGenerator.cs`、`GlobeCellData.cs`、`GlobeCellView.cs`）：
  - 二十面体 → 细分 → 归一化 → Icosphere → 对偶网格。细分级别 3 生成 642 个格子，正好 12 个五边形。
  - 每个格子一个 GameObject，带 `MeshFilter` / `MeshRenderer` / `MeshCollider`；纯数据放在 `GlobeCellData`；单格颜色用 `MaterialPropertyBlock`。
  - 多边形角点在每个格子的切平面内排序，避免格子自交。
  - 拓扑校验日志：五边形数量、邻居数、双向邻接。
- **三层可视化**（`GlobeDisplayMode`，运行时可切换）：
  - `BaseSphere`：光滑底球。
  - `IcosphereTriangles`：底球加亮色三角边线。
  - `DualCells`：六边形/五边形格子加暗色边框线。
- **编辑器内生成**（`Assets/Editor/GlobeGeneratorEditor.cs`）：Inspector 中的 Generate / Clear 按钮，以及右键上下文菜单项。生成的网格是程序化的、未保存为资源，因此重新打开场景后需要重新执行 Generate。
